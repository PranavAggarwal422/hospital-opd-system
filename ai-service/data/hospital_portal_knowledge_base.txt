[PLATFORM_OVERVIEW]

The Hospital Portal is an online OPD management platform designed to help patients manage hospital-related activities digitally.

The platform allows patients to:
- search hospitals
- explore hospital departments
- view doctor sessions
- understand OPD schedules
- book appointments
- manage appointments
- track token numbers
- access reports
- download reports
- manage account/profile settings

Main sections of the portal:
- Dashboard
- Book Appointment
- My Appointments
- Reports
- Account/Profile

The chatbot acts as a healthcare navigation and hospital guidance assistant.

The chatbot can:
- explain workflows
- help users navigate the portal
- explain appointment processes
- explain hospital rules and policies
- help identify suitable departments
- explain report terminology in simple language

The chatbot cannot:
- directly book appointments
- directly cancel appointments
- modify database records
- update user accounts
- prescribe medicines
- provide confirmed diagnosis
- replace licensed healthcare professionals

Common user questions:
- What can I do on this portal?
- How does this hospital portal work?
- Can chatbot book appointment for me?
- Can chatbot cancel my appointment?
- What services are available online?

[REGISTRATION_WORKFLOW]

New users can create an account using the Registration page.

Users may register using:
- phone number only
- email only
- both email and phone number

At least ONE of the following is required:
- email
- phone number

Registration typically requires:
- full name
- password
- gender

Optional information may include:
- address
- date of birth

Important rules:
- email must be unique
- phone number must be unique
- duplicate accounts using same phone/email are not allowed

Registration may fail if:
- email already exists
- phone number already exists
- required information is missing

After successful registration:
- account is created
- user can login using registered credentials

[APPOINTMENT_VALIDATION_RULES]

Appointment booking follows multiple validation rules.

Users cannot:
- book appointments in the past
- book appointments beyond the next 30 days
- book appointments for inactive hospitals
- book appointments for inactive departments
- book appointments for inactive sessions
- book appointments for inactive doctors

Appointments must match correct session weekday.

Example:
If session is available only on Monday,
users cannot book appointment for Tuesday.

Same-day booking may fail if:
- session timing already ended
- OPD session closed

Booking may fail due to:
- invalid date
- inactive doctor
- inactive session
- hospital restrictions
- incorrect weekday

Common user questions:
- Why is booking failing?
- Why can’t I book appointment?
- Why is selected date invalid?
- Why can’t I book for next month?
- Why can’t I select this doctor?
- Why does portal say session unavailable?

[MULTIPLE_APPOINTMENT_RESTRICTIONS]

Portal prevents duplicate or conflicting appointments.

Users may not be allowed to:
- hold multiple upcoming appointments in same department
- repeatedly rebook cancelled appointments for same day

Example:
If patient already has upcoming Orthopedics appointment,
another Orthopedics booking may be restricted.

Rebooking restrictions may apply after cancellation.

[TOKEN_SYSTEM]

Portal uses token-based OPD queue management.

Important rules:
- token allocation follows FCFS (First Come First Serve)
- earlier bookings usually receive lower token numbers
- lower token numbers usually indicate earlier consultation order

Token numbers are visible in:
- Dashboard
- My Appointments
- Appointment Details

Actual consultation order may vary depending on:
- emergencies
- patient load
- consultation duration
- doctor availability

Common user questions:
- What is token number?
- How is token generated?
- Why is my token high?
- What does FCFS mean?
- How does token system work?
